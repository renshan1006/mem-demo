STEP 02 · 14–24 秒

阻塞规则 + 语义向量 + 规则加权打分

三路融合策略：先分桶避免 O(n²)，再用 Embedding 处理名称相似度，最后规则加权精确判定

① 阻塞分桶 + ② 语义向量 + ③ 规则加权 = 💎 融合匹配
🪣
ROUTE 01

阻塞规则分桶

按 region（客户）/ category（商品）先分桶，把 O(n²) 全量比对降为 O(k × m²)，只在同桶内两两比较。

for region_prefix in groups:
  for pair in bucket:
    yield (record_a, record_b)

// 跨地区/跨品类 → 不可能重复，直接跳过
🧠
ROUTE 02

bge-large-zh 语义向量

基于 SentenceTransformer 加载预训练中文模型，将公司名称 / 商品名转为 1024 维向量，解决中英混合、缩写、同义词的相似度问题。

from sentence_transformers import SentenceTransformer
model = SentenceTransformer('bge-large-zh-v1.5')
vec_a = model.encode(name_a)
vec_b = model.encode(name_b)
sim = cos_sim(vec_a, vec_b) // → [0,1]
⚖️
ROUTE 03

规则加权打分

对税号 / 电话 / SKU / UPC 等结构化字段做精确匹配，地址用编辑距离加权，SKU 冲突会触发惩罚分。

score = (name_sim × 0.45)
    + (tax_id_match × 0.30)
    + (phone_match × 0.15)
    + (addr_sim × 0.10)
    + bonus - penalty
👥
👥

+0.10地址编辑距离
++ bonus税号 / 电话加分项

商品匹配 · 加权公式
0.20名称语义相似度
+0.15品牌语义相似度
+0.10型号语义相似度
+0.25SKU 精确匹配
+0.15UPC 精确匹配
− SKU / UPC 冲突惩罚（上限 0.55，不同 SKU = 不同商品变体）
