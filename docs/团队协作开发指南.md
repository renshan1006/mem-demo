# DataGuardian 团队协作开发指南

---

## 一、项目概述

### 项目名称
**DataGuardian — 多智能体协作的智能数据治理平台**

### 一句话描述
基于多智能体协作架构，自动完成数据质量评估、实体匹配去重、数据标准化等数据治理任务，并用自然语言对话降低使用门槛。

### 技术栈
| 层 | 技术 |
|---|------|
| 智能体决策 | DeepSeek-Chat (Dify Cloud API) |
| 语义理解 | bge-large-zh-v1.5 (SentenceTransformer) |
| 后端框架 | Python 3.9+ |
| 前端界面 | Streamlit |
| 数据存储 | SQLite (开发) / PostgreSQL (生产) |

## 三、接口合约（非常重要！每个人必须遵守）

### 3.1 Agent 统一接口

所有智能体必须实现以下接口，才能被 Registry 调度：

```python
from typing import Dict, Any, List

class BaseAgent:
    """所有智能体的基类"""

    # ---- 必须实现的属性 ----
    name: str          # 智能体唯一名称，如 "entity_match"
    description: str   # 功能描述

    # ---- 必须实现的方法 ----
    def run(self, task: Dict[str, Any]) -> Dict[str, Any]:
        """
        执行任务

        输入 task 结构（所有Agent统一）:
        {
            "action": str,        # 动作名，如 "scan", "match", "normalize"
            "params": dict,       # 动作参数
            "data": pd.DataFrame | None,  # 可选：输入数据
            "context": dict,      # 可选：上下文信息
        }

        输出 result 结构（所有Agent统一）:
        {
            "success": bool,
            "data": Any,          # 核心输出（DataFrame / dict / list）
            "summary": str,       # 一句话结果描述
            "details": dict,      # 详细结果（给前端展示用）
            "error": str | None,  # 错误信息
        }
        """
        raise NotImplementedError

    # ---- 可选实现的方法 ----
    def health_check(self) -> bool:
        """健康检查，返回True表示可用"""
        return True

    def get_actions(self) -> List[str]:
        """返回支持的动作列表"""
        return []
```

### M2 · 组员A · 数据质量评估智能体

**核心文件：**
- `scripts/quality_agent.py` — 新建

**功能：**
- 扫描 CSV 文件，检测数据质量问题
- 调用 Dify LLM 生成中文质量报告
- 提供 Baseline（纯统计）对比

**检测项：**
| 检测项 | Baseline（纯统计） | Agent方法（LLM增强） |
|--------|------------------|-------------------|
| 缺失值 | `df.isnull().sum()` | + 补充建议（如"建议从ERP补全"） |
| 重复行 | `df.duplicated()` | + 模糊去重（名称相似但字段不同） |
| 格式异常 | 正则匹配 | + LLM识别非标准格式 |
| 异常值 | 四分位距（IQR） | + LLM解释异常原因 |
| 一致性 | 值分布统计 | + 跨源字段一致性检查 |

**必须实现的类：**
```python
class DataQualityAgent(BaseAgent):
    name = "data_quality"
    description = "数据质量评估智能体"

    def run(self, task):
        """
        支持的动作:
        - "scan": 扫描指定数据源，返回质量问题列表
        - "report": 生成中文质量报告
        - "compare": 对比多个数据源的质量
        """
```

**交付物：**
- `scripts/quality_agent.py`
- `data/quality_report.json`（示例输出）
- `data/quality_report.md`（示例报告）
- 运行命令：`python scripts/quality_agent.py --source crm_customers.csv`

**核心文件：**
- `scripts/chat_agent.py` — 新建

**功能：**
- 自然语言 → 数据操作：用户说人话，系统自动干活

- 支持多轮对话
- 集成到 Streamlit 界面

**支持的操作示例：**
| 用户输入 | 系统动作 |
|---------|---------|
| "帮我检查 CRM 客户数据质量" | 调用 M2 → 展示报告 |
| "把 ERP 商品地址全部标准化" | 调用 M3 → 展示前后对比 |
| "CRM 里有多少条缺失电话的客户" | 直接查 DataFrame → 回答 |
| "把匹配分数高于 0.8 的客户自动合并" | 调用 M1 → 执行合并 |

**必须实现的类：**
```python
class ChatAgent(BaseAgent):
    name = "chat_assistant"
    description = "对话式数据管理助手"

    def run(self, task):
        """
        支持的动作:
        - "chat": 单轮对话，返回操作+结果+回复
        """
```

**实现思路：**
1. 收到用户自然语言输入
2. 调用 LLM 识别意图 → 输出 `{agent, action, params}`
3. 通过 Registry 调用对应 Agent
4. 用 LLM 将结果转成自然语言回复

**交付物：**
- `scripts/chat_agent.py`
- 在 `streamlit_app.py` 中新增 Chat 界面 Tab

## 七、Baseline 说明（所有人必须理解）

### 什么是 Baseline？
Baseline 是"不做任何智能处理"的原始方法，用来证明你的智能方法确实更好。

### 每个模块的 Baseline

| 模块 | Baseline | 你的方法 |
|------|---------|---------|
| M1 实体匹配 | `difflib` 编辑距离 | bge-large-zh + Dify LLM |
| M2 质量评估 | `df.describe()` + `df.isnull().sum()` | LLM分析 + 中文报告 |
| M3 数据标准化 | 正则表达式替换 | LLM语义理解 + 规则混合 |
| M4 对话助手 | 无（新功能） | NL意图识别 + 多Agent调度 |

### 答辩时怎么讲？
"我们用纯规则/纯统计做Baseline，准确率只有XX%；加上智能Agent后，准确率提升到XX%。这证明了Agent方法的有效性。"

## 八、常见问题

**Q: 我没法访问 Dify API 怎么办？**
A: 先在代码里做好 API 调用的 try/except，失败了就返回降级结果。`dify_client.py` 里有现成的降级逻辑可以参考。

**Q: 我的 Agent 怎么调另一个 Agent？**
A: 统一通过 Registry：`registry.call("agent_name", task)`。不要直接 import 别人的模块。

**Q: 前端界面怎么写？**
A: 参考现有 `streamlit_app.py` 的结构，每人在 `/docs` 下写自己模块的前端代码片段，组长统一合并。

**Q: 数据是合成的不够"真实"？**
A: 合成数据有 6200 条，设计时已经模拟了真实数据质量问题（中英混用、格式差异、地址变异）。答辩时可以说"系统架构允许替换真实数据即可"。
