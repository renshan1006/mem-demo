# 智能体设计文档

## 1. 目标与范围

构建基于 Dify 的主数据管理（MDM）智能体，实现跨源主数据接入、多策略实体匹配、人工审核流转、黄金记录生成与阈值反馈优化的全流程闭环。

## 2. 系统架构

### 2.1 整体架构图

```
┌─────────────────────────────────────────────────────────┐
│                      数据层                               │
│  CRM/ERP/ECommerce CSV  →  PostgreSQL (黄金记录)           │
│  候选匹配 CSV             →  Redis (Embedding 缓存)         │
└──────────────────────┬──────────────────────────────────┘
                       │
┌──────────────────────▼──────────────────────────────────┐
│                    处理层 (Python)                        │
│  generate_dataset.py    数据生成 (Faker, 6,200 条)        │
│  mdm_pipeline.py        客户匹配管道 (阻塞+Embedding+规则)  │
│  product_pipeline.py    商品匹配管道 (+ SKU/UPC 惩罚)      │
│  embedder.py            bge-large-zh-v1.5 语义向量化       │
│  tune_thresholds.py     阈值回训 (4,356 组合扫描)          │
│  finalize.py            黄金记录生成 (UnionFind 聚类)      │
│  dify_client.py / dify_enrich.py  Dify API 集成           │
│  streamlit_app.py       人工审核界面 (5-Tab)              │
└──────────────────────┬──────────────────────────────────┘
                       │
┌──────────────────────▼──────────────────────────────────┐
│                   智能层 (Dify Cloud)                     │
│  MDM-Entity-Matching-Agent 工作流                        │
│  ├─ Start:     接收两条记录 + entity_type                 │
│  ├─ LLM Node:  语义对比 + 规则判断 + 决策                  │
│  ├─ Code Node:  解析 JSON，标准化输出                      │
│  └─ End:       {decision, confidence, reasoning}         │
│                                                          │
│  模型: DeepSeek-Chat / GPT-4o-mini (可替换)               │
│  Prompt:  客户/商品双套模板，含 Jinja2 条件分支             │
└─────────────────────────────────────────────────────────┘
```

### 2.2 分层说明

| 层 | 技术栈 | 职责 |
|----|--------|------|
| 数据层 | CSV → PostgreSQL + Redis | 持久化黄金记录，缓存 Embedding 向量 |
| 处理层 | Python (pandas, numpy, torch) | 数据生成、预处理、规则打分、Embedding、阈值优化 |
| 智能层 | Dify Cloud (LLM + 工作流) | 模糊匹配对的二次判断、Prompt 辅助决策 |
| 展示层 | Streamlit | 审核仪表板、详情对比、历史管理、数据导出 |

## 3. 匹配工作流

### 3.1 整体流程

```
[数据加载] → [文本规范化] → [Embedding 预计算]
     → [阻塞分桶 (region/category)] → [候选对生成]
     → [多策略打分] → [阈值分类] → [审核队列输出]
     → [人工审核 (Streamlit)] → [黄金记录合并]
     → [阈值回训 (tune_thresholds.py)] → [反馈至下一轮]
```

### 3.2 多策略打分算法

**客户匹配** (权重可配置):
```
总分 = Embedding名称相似度 × 0.45 + 税号精确匹配 × 0.30
     + 电话精确匹配 × 0.15 + 地址编辑距离 × 0.10
     + 税号匹配加分(0.10) + 电话匹配加分(0.05)
```

**商品匹配** (权重可配置):
```
总分 = Embedding名称 × 0.20 + Embedding品牌 × 0.15 + Embedding型号 × 0.10
     + SKU精确匹配 × 0.25 + UPC精确匹配 × 0.15 + 规格编辑距离 × 0.05
     + SKU/UPC匹配加分(0.05) + (品牌∧型号)加分(0.03)
     - SKU不匹配惩罚: 上限 0.55
     - UPC不匹配惩罚: 上限 0.55
```

### 3.3 决策分类

| 区间 | 默认阈值 | 优化后（客户） | 优化后（商品） | 动作 |
|------|:------:|:-----------:|:-----------:|------|
| auto_merge | ≥ 0.80 | ≥ 0.53 | ≥ 0.54 | 自动合并 |
| review | 0.50-0.80 | 0.50-0.53 | 0.51-0.54 | 人工审核 + Dify LLM |
| no_match | < 0.50 | < 0.50 | < 0.51 | 判定为不同实体 |

### 3.4 Dify LLM 增强（审核队列二次判断）

```
审核队列候选对 → dify_enrich.py → Dify Workflow API
  → LLM 语义对比 + 规则推理 → {decision, confidence, reasoning}
  → 写入 customer_review_enriched.csv
  → Streamlit 审核界面展示 Dify 建议
```

## 4. Dify 工作流编排

### 4.1 节点拓扑

```
Start (输入) ──→ LLM Node (语义匹配) ──→ Code Node (解析) ──→ End (输出)
```

### 4.2 节点说明

| 节点 | 类型 | 功能 |
|------|------|------|
| Start | 输入 | 接收 entity_type、记录 A/B 的全部字段（支持客户/商品双模式） |
| LLM 语义匹配 | LLM | 加载 System Prompt + Jinja2 条件 User Prompt，调用 LLM 推理 |
| Code 解析 | Python | 正则清理 LLM 输出，json.loads 解析，标准化 decision 字段 |
| End | 输出 | 返回 `{decision, confidence, reasoning, signals}` |

### 4.3 Prompt 设计

**System Prompt** 核心要素：
- 角色设定：MDM 专家
- 客户规则：税号 > 电话 > 名称语义 > 地址 > 地区
- 商品规则：SKU > UPC > 名称语义+品牌+型号 > 规格 > 品类
- 置信度阈值：≥0.85 merge, 0.65-0.84 review, <0.65 no_match
- 输出格式：严格 JSON（禁止 markdown 代码块）

**User Prompt** 使用 Jinja2 条件分支：
```
{% if entity_type == "customer" %}
  [客户字段列表]
{% else %}
  [商品字段列表]
{% endif %}
```

### 4.4 模型配置

| 参数 | 值 | 说明 |
|------|-----|------|
| model | deepseek-chat | 性价比高，中文能力强 |
| temperature | 0.1 | 低温度保证输出一致性 |
| provider | openai_api_compatible | 兼容 OpenAI 协议 |

## 7. 关键决策点

| 决策点 | 方案 | 依据 |
|--------|------|------|
| Embedding 模型选型 | bge-large-zh-v1.5 | 中文 C-MTEB 榜单 Top3, 1024 维向量 |
| 名称相似度混合比 | Embedding×0.8 + 编辑距离×0.2 | 平滑处理生僻词 |
| 商品 SKU 冲突惩罚 | 上限 0.55 | 不同 SKU = 不同商品变体 |
| auto_merge 阈值 | 扫描 4,356 组合取最优 | 精确率≥95%, 最小化审核量 |
| 阻塞键 | region(客户) / category(商品) | 跨地区/跨品类不可能重复 |
| Dify vs 本地规则分工 | Dify 做模糊二审 | 降低成本（只调审查队列的 272 条） |
